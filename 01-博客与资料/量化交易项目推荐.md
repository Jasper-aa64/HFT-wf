# 量化交易项目推荐

## 先给结论

最适合你的组合是：

```text
C++ 核心：Udaynanda/cpp-trader-backtester
Agent 闭环：AgentQuant
LLM 自动改策略思路：Auto-Quant
```

**不要找一个大而全项目直接改**，而是选一个 C++ 回测核心，再参考 AgentQuant / Auto-Quant 的闭环，把两者接起来。这样最能学到东西。

---

## 1. cpp-trader-backtester：最适合你练 C++ 核心

**链接：** https://github.com/Udaynanda/cpp-trader-backtester

这个项目技术含量主要在 **C++ 量化系统底层**，不是 AI。它有事件驱动架构、可插拔策略、价格时间优先订单簿、内存池、market/limit order、部分成交、SIMD 优化这些东西。README 里还写了性能指标，比如 34M ticks/sec、7.65M orders/sec matching、sub-microsecond latency。

**适合你学这些：**

```text
1. C++20 工程结构
2. 事件驱动回测
3. Tick 级行情处理
4. 订单簿撮合
5. Market order / limit order
6. 部分成交
7. 内存池
8. 延迟和吞吐性能测试
9. 策略接口设计
```

**缺点也要看清楚：** 目前仓库很小，只有 2 commits，star 也很少。所以它不像成熟框架，更像一个**小而硬核的 C++ 回测样板**。

**评价：**

| 维度 | 评分 |
|------|------|
| 技术含量 | 高 |
| 学习价值 | 高 |
| 直接可用性 | 中 |
| 适合你程度 | **最高** |

如果你主要想练 C++，这个最值得先看。

---

## 2. AgentQuant：最适合你学 Agent 闭环

**链接：** https://github.com/OnePunchMonk/AgentQuant

AgentQuant 的技术含量不在 C++，而在 **Agent 如何做量化研究闭环**。它不是简单写个 prompt，而是一个 analyze → hypothesize → backtest → reflect → store 的循环，还用了 SQLite memory 保存历史结果。

**比较值得学的点：**

```text
1. ReAct agent loop
2. 市场状态识别
3. LLM → Grid Search → Random fallback
4. 回测锦标赛
5. Sharpe / Calmar / Sortino / max drawdown 指标
6. 结果反思
7. SQLite 实验记忆
8. 参数网格约束
9. 防止 look-ahead bias
10. 测试体系
```

README 里明确写了，Agent 会分析 VIX 分位数、多周期动量、SMA 趋势信号，然后通过 LLM、网格搜索、随机 fallback 生成参数，回测后计算 Sharpe、Calmar、Sortino、最大回撤、bootstrapped Sharpe，再把最好结果存进 SQLite memory。

而且它有测试，包括 config、metrics、regime、features、strategies、backtest、proposal_generator 等模块。

**评价：**

| 维度 | 评分 |
|------|------|
| 技术含量 | 高 |
| 学习价值 | **很高** |
| C++ 价值 | 低 |
| Agent 价值 | **最高** |
| 适合你程度 | 作为 Agent 参考非常合适 |

你可以不直接用它的策略，但可以照着它的闭环设计，把它改成调用你的 C++ 回测程序。

---

## 3. Auto-Quant：最适合理解"LLM 自动改策略"的思路

**链接：** https://github.com/TraderAlice/Auto-Quant

Auto-Quant 的思路非常清楚：给 LLM Agent 一个 FreqTrade 回测环境和一个策略文件，让 Agent 修改策略、运行回测、判断结果有没有提升、保留或丢弃，然后重复。它自己也说成功标准不是找到赚钱策略，而是闭环能不能跑起来并产出可解释的 `results.tsv`。

**适合你学：**

```text
1. Agent 如何改策略文件
2. 如何跑 backtest
3. 如何判断改进
4. 如何保留 / 丢弃结果
5. 如何做实验记录
6. 如何避免只看假高 Sharpe
```

**缺点：** 偏 Python / FreqTrade，不是 C++ 项目。

**评价：**

| 维度 | 评分 |
|------|------|
| 技术含量 | 中高 |
| 学习价值 | 高 |
| C++ 价值 | 低 |
| Agent 思路价值 | **很高** |
| 适合你程度 | 作为思路参考合适 |

你可以把它的逻辑换成：

```text
FreqTrade backtest  →  ./cpp_backtester --config xxx.json
```

这样就变成你的 C++ + Agent 项目。

## 4. QuantMuse：架构好看，但不一定最适合你深入学 C++

**链接：** https://github.com/0xemmkty/QuantMuse

QuantMuse 是一个综合量化交易系统，README 说它包含数据收集、策略执行、实时数据处理、因子分析、AI 情绪分析、风险管理等完整 pipeline，并且有 Python layer 和 C++ core engine。

**架构包含：**

```text
Python Layer:
- data fetchers
- strategy framework
- AI/ML
- factor analysis
- backtesting engine
- visualization
- storage
- web UI

C++ Core Engine:
- order execution
- risk management
- portfolio
- data loading
- strategy engine
- performance
```

**适合你参考：**

```text
1. Python + C++ 分层架构
2. AI/ML 和量化系统怎么组织
3. 数据层、策略层、风控层、执行层怎么拆
4. Dashboard 怎么做
5. 风险管理模块怎么设计
```

**问题是：** 可能太"大杂烩"——AI、Web、实时行情、风险、Dashboard、C++ backend 都有。你如果直接从它开始，容易看得很散，不一定能真正深入 C++ 核心。

**评价：**

| 维度 | 评分 |
|------|------|
| 技术含量 | 中高 |
| 学习价值 | 中高 |
| C++ 深度 | 中 |
| 架构参考价值 | 高 |
| 适合你程度 | 可以看，不建议作为第一主线 |

## 5. FinceptTerminal：最强但不适合你现在上手

**链接：** https://github.com/Fincept-Corporation/FinceptTerminal

FinceptTerminal 看起来最牛。它是 C++20 / Qt6 原生金融智能平台，贡献文档里写了 50+ screens、embedded Python analytics、100+ data connectors。技术栈包括 C++20、Qt 6.8.3、Qt Charts、Qt Network、Qt WebSockets、SQLite、Embedded Python 3.11.9、CMake、Ninja 等。

它技术含量确实高，能学到：

```text
1. C++20 大型桌面应用
2. Qt6 GUI
3. 金融终端架构
4. Python 嵌入式分析
5. 数据连接器
6. 图表、网络、数据库
7. 模块化桌面软件
```

**但问题也很明显：**

```text
1. 太大
2. Qt6 学习成本高
3. 金融终端方向重于回测引擎
4. 你容易陷入界面和依赖环境
5. 不适合作为第一个 C++ 量化 + Agent 项目
```

**评价：**

| 维度 | 评分 |
|------|------|
| 技术含量 | **很高** |
| 学习价值 | 高，但容易迷路 |
| C++ 深度 | 高 |
| Agent 可参考性 | 中 |
| 适合你程度 | 只建议看架构，不建议直接改 |

它更适合你后期看，不适合现在一上来就 fork 改。

## 6. madigan：适合学强化学习交易，但不适合你当前方向

**链接：** https://github.com/Hanwant/madigan

madigan 是强化学习交易项目，它把交易问题形式化为 MDP：agent 和 environment 交互，动作空间是 buy/sell/hold 或目标组合，reward 是 equity returns / Sharpe / Sortino / transaction costs。

它的 Environment 部分写了 C++ with bindings to Python，目的是速度，但也说明目前只有最小 accounting 功能，order semantics pending。

**适合你学：**

```text
1. 强化学习交易环境
2. MDP 建模
3. reward shaping
4. C++ environment + Python binding
5. Sharpe / Sortino reward
```

**但：** 它不是 LLM Agent，也不是传统 C++ 回测系统主线。

**评价：**

| 维度 | 评分 |
|------|------|
| 技术含量 | 中高 |
| 学习价值 | 中高 |
| C++ 价值 | 中 |
| Agent 价值 | 偏 RL，不是 LLM |
| 适合你程度 | 暂时不建议 |

除非你明确想做强化学习交易，否则先别碰它。

## 你真正应该做的项目形态

不是"用某一个项目"，而是做一个小项目：

```text
CppQuantAgent
```

**核心结构：**

```text
CppQuantAgent/
├── engine_cpp/
│   ├── include/
│   ├── src/
│   ├── strategies/
│   ├── tests/
│   └── CMakeLists.txt
├── agent_py/
│   ├── optimizer.py
│   ├── experiment_memory.py
│   ├── evaluator.py
│   └── prompts/
├── data/
├── reports/
├── experiments/
└── README.md
```

**第一版目标：**

**C++ 部分：**

```text
1. 读取 CSV 行情
2. 跑双均线策略
3. 模拟订单成交
4. 计算资金曲线
5. 输出 result.json
```

**Python Agent 部分：**

```text
1. 读取 result.json
2. 修改 fast / slow 参数
3. 调用 C++ 程序
4. 对比 Sharpe、收益率、最大回撤
5. 保存最优结果
6. 写实验日志
```

**命令示例：**

```bash
./quant_backtester --data data/aapl.csv --strategy ma_cross --fast 5 --slow 20
```

**输出示例：**

```json
{
  "strategy": "ma_cross",
  "fast": 5,
  "slow": 20,
  "total_return": 0.184,
  "max_drawdown": 0.072,
  "sharpe": 1.36,
  "trades": 24
}
```

**Agent 优化循环：**

```text
第 1 轮：fast=5, slow=20，Sharpe=1.36，保留
第 2 轮：fast=10, slow=30，Sharpe=1.12，丢弃
第 3 轮：fast=8, slow=25，Sharpe=1.51，保留
```

# 论文实验证据计划

Updated: 2026-05-26

## 0. 写作原则

这篇论文/技术报告必须 evidence-first。

任何结论进入正文前，都要能回答：

- 实验在哪台机器、哪个 commit、哪个 run directory 上跑的？
- baseline 和 candidate 是什么？
- correctness gate 是否通过？
- timing / jitter / host health artifact 在哪里？
- 结论是 clean evidence、noisy evidence，还是 invalid evidence？
- 这个证据支持的是 discovery、diagnosis、promotion review，还是 final acceptance？

不能写成结论的内容：

- 只来自论文启发、没有本项目实验的判断。
- 只来自 agent 口头分析、没有 artifact 的判断。
- noisy timing 下的自动 accepted。
- 未实现的 Duet / stage instrumentation 方案。

## 2. 论文需要补齐的关键实验

### E1. Host jitter audit reproducibility

目标：

验证 host weather station 能区分可用窗口和不可用窗口。

实验设计：

- Quiet window: control-only run, `measure-runs=5` or more。
- Busy window: intentionally detect active runner and block, not run timing。
- Artifact:
  - `host_readiness.json`
  - `host_jitter_samples.tsv`
  - `host_jitter_summary.json`
  - `psi_host_jitter_audit_summary.json`

通过标准：

- Snapshot-only host check must not produce `QUIET`。
- Active runner must block promotion timing。
- Control-only samples must be recorded and summarized。

### E2. Validation lock v2

目标：

证明不是只靠人肉约束，而是 harness 可以阻止并发性能验证互相污染。

实验设计：

- First run acquires `/root/work/.perf_validation.lock`。
- Second performance timing run attempts to start and must fail before timing。
- Release only succeeds for the owning run id。

Artifact:

- lock metadata JSON
- blocked-run summary
- unit tests for acquire / block / release / non-owner release refusal

通过标准：

- No second timing process starts while lock is held。
- Blocked run is recorded as infra-blocked, not candidate failure。

### E3. Stack m24 promotion review

目标：

验证 `stack_skip_unused_row_fields` 是否能从 noisy discovery 进入 clean promotion evidence。

前置条件：

- validation lock held, or strict manual exclusivity
- no active `PsiTraderRunner`
- host audit is `QUIET`

通过标准：

- build pass
- compare pass
- paired sample count: 24
- noise flag clean enough for promotion gate
- no mid-run contamination

结果解释：

- clean accepted: still requires manual risk review before Code1 promotion
- strong positive but noisy: keep `PROMOTION_CANDIDATE_UNDER_NOISY_HOST`
- compare/build fail: reject or classify by failure type

### E4. Micro-patch vs stack effect-size comparison

目标：

把“组合 patch 提高信号量”从直觉变成表格。

实验对象：

- micro candidates with compare-pass positive-noisy evidence
- `stack_skip_unused_row_fields`

指标：

- median delta
- CI width
- paired sample count
- noise flag
- verdict

注意：

这不是 acceptance 实验，而是 discovery-process evidence。

### E5. Optional stage / Duet-inspired instrumentation

目标：

只在 Psi stack promotion 问题解决后再做。

可接受范围：

- fixed whitelist probe anchors
- controlled stage timing artifacts
- base/head symmetric measurement where feasible
- diagnosis only

禁止范围：

- agent 自由插 C++ probe
- 插桩结果直接决定 accepted
- 未隔离 runner 的并发 duet timing

## 4. 写作边界


```text
Noisy positive results are retained as promotion candidates under noisy host, not accepted.
```

可以写：

```text
Composite work-reduction patches increase observed effect size, but require clean promotion evidence.
```


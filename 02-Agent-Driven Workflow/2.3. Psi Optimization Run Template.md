## Verdict

```text
PASS / FAIL_BUILD / FAIL_CORRECTNESS / FAIL_PERF / DIAGNOSTIC_ONLY
```

Decision:

```text
KEEP / REVERT / INVESTIGATE
```

Profile-only runs must use `DIAGNOSTIC_ONLY`; do not record them as
`PASS` or `FAIL_PERF`. Actual candidate acceptance is measured against the
current control baseline recorded in the run's control-loop artifacts. Do not
hard-code the active control head, median, or derived thresholds in this
template; record those values in attempts.tsv, performance optimization reports, or run artifacts.

Class A still requires correctness and bundle drift audit. The label only means
one noisy perf sample should not reject the candidate by itself.
